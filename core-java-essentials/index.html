Part 1: The Basics

1. Platform Independence

Java is "Write Once, Run Anywhere" (WORA).

  • JDK (Java Development Kit): Compiles .java into .class (Bytecode).
  • JRE (Java Runtime Environment): Contains standard libraries to run Java apps.
  • JVM (Java Virtual Machine): Reads bytecode and converts it to machine-specific instructions. JVM is platform-dependent!

2. Data Types & Variables

Java is strictly statically-typed, which means that all variables must first be declared before they can be used. This involves stating the variable's type and name:

int gear = 1;

Doing so tells your program that a field named "gear" exists, holds numerical data, and has an initial value of "1". A variable's data type determines the values it may contain, plus the operations that may be performed on it.

Deep Dive: The 8 Primitive Data Types
  • byte (8-bit signed): Min -128, max 127. Useful for saving memory in large arrays. The limit can also serve as a form of documentation.
  • short (16-bit signed): Min -32,768, max 32,767. Useful for memory savings in large arrays.
  • int (32-bit signed): Min -231, max 231-1. In Java SE 8+, represent unsigned 32-bit int (0 to 232-1) using the Integer class (methods like compareUnsigned, divideUnsigned).
  • long (64-bit signed): Min -263, max 263-1. Use when wider range is needed. SE 8+ supports unsigned 64-bit via Long class.
  • float (32-bit IEEE 754): Single-precision. Save memory in large arrays. Never use for precise values like currency! (Use java.math.BigDecimal instead).
  • double (64-bit IEEE 754): Double-precision. Default choice for decimal values. Never use for precise values like currency!
  • boolean: true or false. One bit of information, but "size" isn't precisely defined.
  • char (16-bit Unicode): Single character from '\u0000' (0) to '\uffff' (65,535).

Note on Strings: The java.lang.String class provides special support. String objects are immutable (values cannot be changed once created). While not a primitive, you'll often think of it as one due to special language support.

3. Literals Deep Dive

A literal is the source code representation of a fixed value; literals are represented directly in your code without requiring computation. Primitive types avoid the new keyword entirely.

Integer Literals

An integer literal is of type long if it ends with the letter L or l; otherwise it is of type int. Recommendation: Use the upper case letter L because the lower case letter l is hard to distinguish from the digit 1.

Values of the integral types byte, short, int, and long can be created from int literals. Values of type long that exceed the range of int can be created from long literals. Integer literals can be expressed by these number systems:

  • Decimal (Base 10): Standard digits 0-9.
  • Hexadecimal (Base 16): Digits 0-9 and A-F. Prefix 0x.
  • Binary (Base 2): Digits 0 and 1. Prefix 0b.
// The number 26 int decVal = 26; // Decimal int hexVal = 0x1a; // Hexadecimal int binVal = 0b11010; // Binary
Floating-Point Literals

A floating-point literal is of type float if it ends with the letter F or f; otherwise its type is double (optionally ends with D or d).

Floating point types can also be expressed using E or e for scientific notation.

double d1 = 123.4; double d2 = 1.234e2; // Same as d1, but in scientific notation float f1 = 123.4f;
Character and String Literals

Literals of types char and String may contain any Unicode (UTF-16) characters. Use 'single quotes' for char and "double quotes" for String.

Unicode Escapes: '\u0108' (capital C with circumflex) or "S\u00ED Se\u00F1or" (Sí Señor). Note: Unicode escape sequences may be used elsewhere in a program (such as in field names), not just in literals.

Special Escape Sequences:

  • \b (backspace), \t (tab), \n (line feed)
  • \f (form feed), \r (carriage return)
  • \" (double quote), \' (single quote), \\ (backslash)

Special Literals:

  • null: A special literal used as a value for any reference type. It indicates that some object is unavailable.
  • class literal: Formed by taking a type name and appending .class (e.g., String.class). This refers to the Object (of type Class) that represents the type itself.
Using Underscore Characters

In Java 7+, underscores (_) can appear between digits to separate groups (like commas) and improve readability. This is purely visual and ignored by the compiler.

long creditCardNumber = 1234_5678_9012_3456L; long socialSecurityNumber = 999_99_9999L; float pi = 3.14_15F; long hexBytes = 0xFF_EC_DE_5E; long hexWords = 0xCAFE_BABE; long maxLong = 0x7fff_ffff_ffff_ffffL; byte nybbles = 0b0010_0101; long bytes = 0b11010010_01101001_10010100_10010010;

Invalid Placement Examples:

  • float pi1 = 3_.1415F; // Adjacent to decimal point
  • float pi2 = 3._1415F; // Adjacent to decimal point
  • long ssn = 999_99_9999_L; // Prior to L suffix
  • int x = 52_; // At the end of a literal
  • int x = 0_x52; // In the 0x radix prefix

Valid Placement Examples:

int x1 = 5_2; // OK int x3 = 5_______2; // OK (any number of underscores) int x6 = 0x5_2; // OK (hexadecimal literal)

4. Type Casting (Conversion)

Type casting is used to convert one data type to another. There are two main types:

  • Widening Type Casting (Implicit): Automatically done by the compiler when converting a smaller type to a larger type.

Hierarchy: byte → short → char → int → long → float → double

int num1 = 5004; double num2 = 2.5; // Implicitly casts int to double double sum = num1 + num2; System.out.println(sum); // 5006.5
  • Narrowing Type Casting (Explicit): Done manually by the programmer when converting a larger type to a smaller type.
double doubleNum = 5004.0; // Explicit cast using (targetType) int convertedInt = (int) doubleNum;
Common Error: Type Mismatch

A compilation error occurs if you try to assign a larger data type to a smaller one without explicit casting.

int num1 = 5004; double num2 = 2.5; int sum = num1 + num2; // Error: cannot convert from double to int

Part 2: Control Flow

1. Switch Statement Mastery

The switch statement is a multi-way decision statement that provides a cleaner alternative to long if-else-if ladders. It executes different blocks based on the value of an expression.

  • Supported Types: byte, short, char, int, enum, String (Java 7+), and Wrapper classes (Integer, Short, Byte, Character). Note: long is not supported.
  • Benefits: Improves code readability and structure when handling multiple discrete conditions.
switch(expression) { case value1: // Statements break; // Optional but recommended case value2: // Statements break; default: // Executes if no match found }
Core Rules for Switch Statements
  • Constants Only: Case labels must be constant expressions or literals. Use of variables is Illegal.
  • No Duplicates: Duplicate case values are strictly prohibited.
  • Fall-through: Java switch is a "fall-through" statement. If break is omitted, execution continues into the next case automatically.
  • Default Case: Optional, but executes if no other case matches. It can appear anywhere but is best kept at the end.
Advanced Feature: Enums in Switch

Enums represent a fixed set of constants (e.g., Days of the week, HTTP status codes). Using them in switch statements provides superior type safety and readability compared to integers or strings.

public enum Day { Sun, Mon, Tue, Wed, Thu, Fri, Sat } // Switch handles enum constants directly switch (day) { case Sun: System.out.println("Rest day!"); break; case Mon: System.out.println("Back to work."); break; }
  • Type Safety: You cannot pass an invalid value into the switch; only values defined in the Enum are allowed.
  • Iteration: You can combine enums with loops (e.g., Day.values()) to iterate through all constants and process them via switch.
Advanced Feature: Wrapper Classes in Switch

Java allows the use of **Wrapper classes** (Integer, Short, Byte, Long, and Character) in switch statements. This is possible due to Autounboxing, where the JVM extracts the primitive value automatically.

Integer age = 25; // Wrapper class object switch (age) { case 25: System.out.println("Matching primitive 25."); break; default: System.out.println("No match."); }

Character Wrapper Example:

Character ch = 'c'; switch (ch) { case 'a': System.out.println("Match 'a'"); break; case 'c': System.out.println("Match 'c'"); break; }

Internal Logic: Regardless of placement, the default case only executes if no other conditions match. The wrapper object is unboxed to its primitive counterpart before comparison.

Advanced Scenario: Nested Switch Statements

You can use a switch as part of the statement sequence of an outer switch. Since a switch statement defines its own block, no conflicts arise between the case constants in the inner and outer switches.

switch (year) { case 1: System.out.println("Freshman Basics"); break; case 2: switch (Branch) { // Nested Switch case "CSE": System.out.println("Machine Learning"); break; case "ECE": System.out.println("Antenna Eng."); break; } break; }

2. Loops & Iteration Architecture

Loops are used to execute a block of code multiple times. In Java, execution is sequential by default, and loops provide branching for repeated complexity.

Sr.NoLoop TypeDescription
1while loopRepeats while condition is true. Tests condition before body execution.
2for loopBest for fixed iterations. Compactly manages initialization, condition, and increment.
3do...whileGuarantees at least one execution. Tests condition at the end.
4Enhanced forTraverses arrays/collections directly without index variables.
Loop Code Examples

Standard While Loop:

int i = 1; while (i <= 5) { System.out.println("Count: " + i); i++; }

Standard For Loop:

for (int j = 1; j <= 5; j++) { System.out.println("Iteration: " + j); }

Do-While Loop (Executes at least once):

int k = 1; do { System.out.println("Do-While: " + k); k++; } while (k <= 5);

Enhanced For-Loop (for-each):

int[] numbers = {10, 20, 30, 40}; for (int num : numbers) { System.out.println("Value: " + num); }

Break & Continue Logic:

for (int n = 1; n <= 10; n++) { if (n == 3) continue; // Skip 3 if (n == 7) break; // Stop at 7 System.out.print(n + " "); } // Output: 1 2 4 5 6

3. Loop Control Statements

  • break: Terminates the nearest loop or switch immediately and transfers control to the next statement.
  • continue: Skips the remaining lines in the current loop body and immediately re-tests the loop condition.

Part 3: Packages & Access Control

1. Java Packages Deep Dive

Packages are used in Java to group related classes, interfaces, and sub-packages. They help organize large projects and preclude naming conflicts.

Creating a Package

To create a package, use the package statement. Important: It must be the very first line of code in your file.

package com.baeldung.packages; public class Application { ... }

Disadvantages of Default (Unnamed) Packages:

  • Loss of package structure benefits and inability to have sub-packages.
  • Types in default packages cannot be imported by other packages.
  • The protected and package-private access scopes become meaningless.
Using Package Members (Imports & FQN)

To use a class like TodoItem from another package, you must import it. If names conflict, use a Fully Qualified Name (FQN).

Sub-package Import:

package com.baeldung.packages; import com.baeldung.packages.domain.TodoItem; // Single import import com.baeldung.packages.domain.*; // Wildcard import

FQN Usage (Conflict Resolution):

public class TodoList { // FQN required if another TodoItem exists elsewhere or during direct access private List<com.baeldung.packages.domain.TodoItem> todoItems; }

Choosing the Right Level (Tips from Experts)
  • Restriction First: Use the most restrictive access level that makes sense. Start with private and expand only with good reason.
  • Avoid Public Fields: Except for constants, public fields (like public String name) are strongly discouraged. They link you to a particular implementation and limit your flexibility.
  • Error Prevention: Choosing correct access levels ensures that errors from misuse by other programmers cannot happen.

Part 4: Exception Handling

1. The Hierarchy

Everything inherits from Throwable.

  • Error: Critical JVM issues (e.g., OutOfMemoryError). Do not try to catch these.
  • Checked Exceptions: Must be handled explicitly via try/catch or throws. Checked at compile time (e.g., IOException).
  • Unchecked Exceptions (Runtime): Programming logic flaws (e.g., NullPointerException, ArithmeticException).
try-catch-finally Block Mechanics
  • try: Code that might throw an exception.
  • catch: Handles the specific exception. You can chain multiple catches.
  • finally: Code that always executes whether an exception occurs or not. Primarily used to close resources.

Part 5: Collections Framework

1. List Interface

Ordered collections that allow duplicate elements.

  • ArrayList: Dynamic array. Fast read access, slow insertions/deletions in the middle.
  • LinkedList: Doubly-linked list. Fast insertions/deletions, slow random access.

2. Set Interface

Collections that forbid duplicate elements.

  • HashSet: Unordered. Extremely fast O(1) operations based on hashing.
  • TreeSet: Sorted order inherently using a Red-Black tree structure. O(log N).

3. Map Interface

Key-Value pairs. Maps are NOT Collections technically, but are part of the framework.

Part 6: Interview Cheatsheet

String vs StringBuilder vs StringBuffer

  • String: Immutable perfectly. Stored in String Constant Pool. Threads-safe intuitively.
  • StringBuilder: Mutable. Extremely fast. Not thread-safe.
  • StringBuffer: Mutable. Thread-safe (Synchronized). Slower operations.

== vs .equals()

  • == Operator: Compares memory addresses (Reference equality). Do both variables point to the same physical object?
  • .equals(): Compares object content logic (Value equality). Are the names/IDs identical inside?

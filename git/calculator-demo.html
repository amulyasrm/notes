Phase 2: Building the Addition Feature

We need to add a Javascript file to handle logic. We strictly follow the Real-World workflow by branching first!

# Safely create and switch to a feature branch
git switch -c feature-addition

# Create a new logic file and add code (using terminal echo for demo purposes)
echo "function add(a,b) { return a + b; }" > calc.js

Phase 3: The Hotfix Crisis (Stash)

You begin writing the subtract function in calc.js.

echo "function subtract(a,b) { return a - " >> calc.js

🚨 EMERGENCY! The main branch production CSS is broken. You need to switch to main immediately and pull the boss's updates to fix it, but your calc.js is half-written and broken!

If you run git switch main now, Git will block you because your broken code would bleed into the main branch.

# 1. Hide our half-written work quickly!
git stash

# 2. Safely switch to main and pull updates
git switch main
git pull origin main

# ... (You quickly fix the CSS bug on main, add, commit, and push) ...

# 3. Switch back to your feature and restore the hidden work
git switch feature-addition
git stash pop

📊 Git State Tracking Details

ActionWorking DirectoryStash "Backpack"
After half-writing code calc.js is Modified (Unstaged) Empty
After `git stash` Clean (Reverted to last commit) Holds a snapshot of the broken calc.js
After `git stash pop` calc.js is Modified again! Empty

Phase 4: Fixing Mistakes

You accidentally generate a massive compiler log file, and you also accidentally delete index.html!

touch debugger.log       # Accidentally created a log file
rm index.html            # Accidentally deleted the HTML
git status

Git Status says: deleted: index.html and Untracked: debugger.log.

# 1. Restore the deleted HTML file from the Staging Area cache
git restore index.html

# 2. Forcibly clean the un-tracked log file we don't want
git clean -fd

To prevent .log files forever, we create a `.gitignore`.

echo "*.log" > .gitignore
git add .
git commit -m "Add subtract logic and gitignore"

Phase 5: The Pull Request & Merge

Our feature branch is complete locally. Now we must put it in the cloud and ask for a team review.

# 1. Inspect our final changes
git log --oneline

# 2. Push our new branch to GitHub
git push origin feature-addition

Action Moves to GitHub (The PR)

We log into GitHub.com and open a Pull Request targeting the main branch. We cannot merge it ourselves. A senior developer reviews calc.js, leaves a comment ("Great logic!"), and clicks Approve.

Then, they click the green Merge Pull Request button. GitHub merges the branch into main.

The Cleanup

Now that the code is merged on GitHub, our local computer is out of date.

# 1. Switch back to the main branch locally
git switch main

# 2. Download the new merged changes from the cloud
git pull origin main

# 3. Clean up our old, now-useless local feature branch
git branch -d feature-addition
